# Project Atlas — Project Constitution

These principles are permanent. Every sprint, engine, and feature must honor them.

## Identity
- **Project Atlas** is a Construction Intelligence Platform.
- It is **not** an ERP, CRM, project-management tool, or reporting system.
- Its purpose: transform everyday construction communication into structured, searchable, trustworthy construction knowledge.

## Platform Flow
`Reality → Memory → Timeline → Intelligence → Operational Intelligence → Execution → New Reality`

## Permanent Architectural Principles
1. **Construction comes first; AI comes second.** Operations must continue even if every AI service is unavailable.
2. **AI never blocks workflows.** Event capture returns to the user in <300 ms. AI runs asynchronously.
3. **Raw data is preserved permanently.** Voice recordings, photos, documents, timestamps, GPS, user, metadata — never discarded.
4. **AI interpretations are stored separately from factual records.** Original facts are immutable.
5. **Human corrections never overwrite original facts.** Corrections are linked records, append-only.
6. **Every AI insight must be explainable through Evidence.** No conclusion without supporting artefacts.
7. **Every architectural component is independently replaceable.** Single-responsibility engines.
8. **Data quality takes precedence over AI sophistication.**
9. **Platform architecture takes precedence over feature expansion.**
10. **Human judgement is authoritative.** AI proposes; humans accept, edit, or reject.

## Operational Accountability Principle (V3)
Every Operational Item must always clearly answer three questions:
- **Why does this exist?**
- **Who currently owns it?**
- **What is preventing completion?**

These answers must be visible without opening additional screens.

## What Project Atlas Will Never Be
Procurement · Inventory · BOQ · Billing · Accounting · CRM · Payroll · Scheduling · Client Portal · Predictive forecasting · Workflow automation (until explicitly approved as a future engine).

These belong to future or external systems, not Project Atlas.
